## 🔴 Не реализовано — требует отдельной сессии

### U1-old — (закрыто выше, оставлено для истории решения)

**Требование Архитектора:** ядро должно РЕГУЛЯРНО спрашивать подключённого
ИИ, актуально ли выданное решение относительно новых технологий, и если
нет — инициировать пересмотр.

**Почему не сделано сейчас:** архитектурное ограничение стателессного HTTP —
ядро не инициирует соединение с агентом само, только отвечает на запросы
агента. "Push"-уведомление в текущей схеме невозможно без смены транспорта
(WebSocket/SSE к агенту, которого сейчас нет).

**Реализуемый путь (для следующей сессии):**
1. При выдаче картриджа (`query.py`) — если `last_tech_check` символа старше
   N дней (например 30), добавлять в ответ флаг `tech_check_requested: true`
   с директивой агенту: "перед использованием проверь актуальность через
   веб-поиск/свои знания, доложи через `/api/v1/hook_reply`"
2. Это переиспользует уже существующий `hook_reply.py` (после N6-фикса он
   уже пишет обновления в `archivist.archive()`) — не нужен новый канал
3. При каждом успешном использовании обновлять `last_tech_check = NOW()`
   в `_process_archive` независимо от similarity-ветки

### U2 — Полный цикл "переговоры с агентом → тест 100% → сохранение как идеальное для условий X"

**Требование Архитектора:** если задача агента — частный случай (личные
предпочтения пользователя агента), ядро соглашается и выдаёт решение под
эти условия, но при подтверждённом 100% результате сохраняет его как
"идеальное решение ПРИ ТАКИХ ОБСТОЯТЕЛЬСТВАХ" (`context_conditions`
заполнен), не как универсальное.

**Что готово:** схема (`is_universal=False` + `context_conditions`) уже
принимает такую запись, `find_symbols` по умолчанию её не покажет другим
пользователям.

**Чего не хватает:** автоматического РАСПОЗНАВАНИЯ, что текущая задача —
частный случай (нет эвристики "это персональное предпочтение, не
универсальный факт"), и явного пути в `_process_archive`/`archive()` чтобы
пометить именно так, а не как обычное universal-решение. Сейчас archive()
всегда пишет `is_universal=True` по умолчанию — нужно:
1. Добавить в `ArchiveRequest`/`archive()` параметр (заполняется флагманом
   или консьержем при обнаружении явно личного контекста в ТЗ)
2. Явно спросить в консьерже ("это решение специфично для ваших личных
   предпочтений или это универсальный факт?") — расширение
   `api/routes/concierge.py`
