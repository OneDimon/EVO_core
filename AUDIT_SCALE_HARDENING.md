## Закрытые проблемы масштабирования

### S1 — ivfflat индекс со статичным lists=100
- [x] **Проблема:** `lists ≈ sqrt(rows)` — при росте базы до миллионов
  символов индекс не пересчитывается автоматически, поиск деградирует.
- [x] **Правка:** `db/migrations/006_hnsw_index.sql` — HNSW вместо ivfflat,
  не требует подбора параметра под размер таблицы.
- [x] **Влияет на:** `db/pg_client.py` (hnsw.ef_search в init-хуке пула).

### S2 — пул соединений БД рассчитан на разработку (min=2, max=10)
- [x] **Проблема:** При тысячах пользователей и нескольких воркерах API
  10 соединений на пул — узкое горлышко.
- [x] **Правка:** `db/pg_client.py::get_pool` — min=5, max=25 на воркер,
  конфигурируемо через `PG_POOL_MIN`/`PG_POOL_MAX`.
- [x] **Влияет на:** `docker-compose.yml` postgres `max_connections=300`
  (запас под workers × pool_max + служебные соединения).

### S3 — API запущен в dev-режиме (--reload, без --workers)
- [x] **Проблема:** Один процесс, авто-рестарт на изменение файлов —
  неприемлемо для прода с параллельной нагрузкой.
- [x] **Правка:** `docker-compose.yml` — `--workers ${UVICORN_WORKERS:-4}`,
  `--reload` убран из основного конфига (для dev — override-файл).

### S4 — блокирующие CPU-bound вызовы в event loop
- [x] **Проблема:** zstd compress/decompress (и train_dictionary) — синхронные
  вызовы внутри async функций блокировали весь event loop воркера на время
  операции — все параллельные запросы вставали в очередь.
- [x] **Правка:** `shards/shard_client.py` — все вызовы `compress`/`decompress`/
  `train_dictionary` обёрнуты в `asyncio.to_thread`.

### S5 — гонка при конкурентной записи похожих решений
- [x] **Проблема:** Два флагмана одновременно решают похожую задачу — оба
  проходят similarity check ДО того, как второй увидит запись первого,
  оба создают почти-дубль вместо Тип А/Б.
- [x] **Правка:** `core/archivist.py::_process_archive` — PostgreSQL
  `pg_advisory_lock` на грубый бакет вектора (округление первых 16
  измерений), сериализует только семантически близкие записи, не
  блокирует несвязанные. Работает между всеми воркерами (DB-level, не
  Python-level Lock).
