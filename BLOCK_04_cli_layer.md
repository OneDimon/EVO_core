# БЛОК 04 — CLI Layer (Скелетонизатор контекста)

**Назначение:** Устанавливается на ПК разработчика. Перехватывает файлы проекта перед отправкой флагману, вырезает тела функций и мусор, оставляет только структуру и сигнатуры. Сжимает контекст с ~44 000 токенов до ~1 300.

## Коннекторы

### Получает на вход
| Источник | Что приходит | Формат |
|----------|-------------|--------|
| Cursor / VS Code / Claude Code | Файлы проекта (сырой код) | Файловая система |
| Разработчик | Конфиг маршрутизации (через LiteLLM) | `config.yaml` |

### Отдаёт на выход
| Получатель | Что отдаёт | Формат |
|-----------|-----------|--------|
| [БЛОК 01](BLOCK_01_core_engine.md) | Очищенный сигнал (скелет кода) | HTTP POST → EVO endpoint |
| Флагман (через LiteLLM прокси) | Сжатый контекст | tokens stream |

## Зависимости

**Должны быть готовы до начала:**
- [БЛОК 01](BLOCK_01_core_engine.md) — ядро запущено и принимает запросы
- [БЛОК 05](BLOCK_05_mcp_server.md) — MCP-сервер готов (CLI передаёт через него команды исполнения)

**Скелетонизация на стороне оболочек:** Cursor, Claude Code, Гермес, OpenClaude делают это сами в соответствии со своими механиками. EVO принимает уже очищенный сигнал. CLI-агент — для кастомных сценариев и прямого подключения без оболочек.

---

## Принцип работы

```
Разработчик отправляет задачу
        │
        ▼
CLI-агент перехватывает файлы проекта
        │
        ▼
Скелетонизация:
  - Убрать тела функций → оставить сигнатуры
  - Убрать комментарии → оставить docstrings ключевых функций
  - Убрать импорты → оставить список модулей
  - Оставить структуру классов / схему БД / конфиги
        │
        ▼
POST /api/v1/query → EVO Core Engine
  + скелет кода
  + запрос пользователя
        │
        ▼
Получить картридж от ядра
        │
        ▼
Передать флагману через LiteLLM прокси:
  картридж + очищенный контекст + запрос
```
