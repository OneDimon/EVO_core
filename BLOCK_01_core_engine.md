## Конвейер обработки (детально)

### Фаза 0 — Минимальный эндпоинт

```
POST /api/v1/query
{
  "user_request": "...",
  "flagship_plan": ["шаг 1", "шаг 2", ...]
}
```

1. Векторизовать `user_request` через модель эмбеддингов
2. Отправить в БЛОК 02: найти top-5 SCL-символов
3. Для каждого символа — отправить в БЛОК 03: декомпрессия ячейки
4. Параллельно разворачивать гиперлинки `[[EVO:SCL-XXXX]]` из тела
5. Собрать картридж (склеить инструкции под шаги плана)
6. Вернуть флагману

```
Response:
{
  "cartridge": "...",          // готовые инструкции
  "symbols_used": ["SCL-..."], // какие символы задействованы
  "scenario": "full|partial|empty" // тип совпадения
}
```

### Фаза 1 — Полный конвейер (добавляется поверх Фазы 0)

**Этап: Интервью флагмана**
- При `scenario: empty` — выдать флагману смежный скилл + задачу: «адаптируй и отчитайся»
- При `scenario: partial` — выдать набор символов под каждый шаг плана отдельно

**Этап: Верификация (БЛОК 06)**
- Перехватить вывод флагмана
- Отправить в YMS-MMM
- При fail × 3 — отправить webhook в [БЛОК 07](BLOCK_07_immune_system.md)

**Этап: Архивация (Тип А / Тип Б)**
- Тип А (улучшение существующего): перезапись ячейки на шарде + рейтинг +1
- Тип Б (новые обстоятельства): создать новый SCL-символ + новая ячейка на шарде
- Переходный маяк в метаданных: «было [X] → стало [Y]»
- Хук-допрос флагману: «...или есть что-то ещё новее?»
- Всё асинхронно — пользователь не ждёт
