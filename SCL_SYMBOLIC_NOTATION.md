## Полная форма символа

```
Φ_{s·n}^{a^m}
```

| Элемент | Обозначение | Смысл |
|---------|-------------|-------|
| **Φ** | Базовый символ | Макро-область из 32 корней |
| **^a** | Надстрочный | Специализация / условие применения |
| **^m** | Степень надстрочного | confirmed_by — кол-во подтверждений |
| **_s** | Подстрочный | Конкретная среда / инструмент / контекст |
| **_n** | Порядковый номер | Уникальный ID внутри подраздела |

---

## Алгебраические операции над символами

### Сложение — ассимиляция знаний из смежных областей

```
β_{s·n}^{a^m} + λ_{k·j}^{b^p} = лигатура
```

Применяется когда знание подтверждено в нескольких независимых областях.
Три подтверждения (confirmed_by = 3) → автоматическое создание лигатуры.

Пример:
```
τ^{auto^2}_{zp} + ε^{pay^2}_{crypto} + κ^{sec^1}_{auth}
= [τ⊕ε⊕κ]^{auto_pay_secure}_{0001}
```
= Технология × Экономика × Кибернетика → комплексное решение авторизации с оплатой

### Умножение — усиление (степень подтверждения)

```
Φ^{a^m}  →  m растёт с каждым независимым подтверждением
```

`τ^{auto^1}` — применено 1 раз в одной области  
`τ^{auto^3}` — подтверждено в 3 независимых областях → кандидат на лигатуру  
`τ^{auto^7}` — высокий R_f, проверено многократно, топ рейтинга

### Разность — эволюция версий (что изменилось)

```
τ^{auto^3}_{zp_0047} - τ^{auto^2}_{zp_0031} = Δ(HSR + крипто)
```

Разность фиксируется в `evolution_note` как переходный маяк:
`было [Genshin + карты] → стало [HSR + крипто-шлюз]`

### Вектор направления — кластер решений

По вектору символа лежит семья решений одной задачи:
```
→ τ^{auto}_{zp} : [0031_legacy, 0047_current, 0052_future]
                    ↑устаревший    ↑актуальный   ↑новейший
```

Вдоль этого вектора: та же задача, разные стеки, разные времена.
Ядро видит весь ландшафт и выдаёт лучшее под конкретные условия.

## Правила уникальности (нарушение = Критическая Ошибка)

**Уникальность достигается точностью описания.** Два похожих решения в разных
условиях = разные символы с разными надстрочными/подстрочными.

```
τ^{auto^2}_{zp_0047}   ← авторизация ZennoPoster HSR + крипто 2026
τ^{auto^1}_{zp_0031}   ← авторизация ZennoPoster Genshin + карты 2024 (is_legacy)
τ^{auto^1}_{flask_0012} ← та же задача но Flask
τ^{auto^1}_{py_0019}   ← та же задача, чистый Python
```

Все четыре — в одном кластере по вектору. Все четыре уникальны по нотации.
Никакой двусмысленности — каждый адрес описывает ровно одно решение
в ровно одном наборе условий.

**Правило "лекарство под задачу":**  
Метаданные символа = краткий диагноз + название лекарства:
- `label` — одна строка: «что за задача + что применяется как решение»
- `science/section/subsection` — классификация в фрактале
- `applicable_stacks[]` — условия применимости
- `evolution_note` — как эволюционировало (разность версий)

## Нотация гиперлинков в теле ячейки

```
[[EVO:τ^{auto}_{zp_0047} | авторизация ZennoPoster HSR]]
[[EVO:ε^{pay}_{crypto_0021}:step_3 | крипто-шлюз без KYC, шаг 3]]
[[EVO:τ^{db}_{pg_0014} | PostgreSQL async pool под нагрузку]]
```

Формат: `[[EVO:символ | краткое описание задачи и лекарства]]`

Локальная модель читает адрес → понимает контекст ссылки без открытия ячейки.
Описание после `|` уточняет зачем эта ссылка в данной инструкции.

## Лигатура: полная форма

```
[Φ₁ ⊕ Φ₂ ⊕ Φ₃]^{специализация^m}_{контекст_номер}
```

Пример:
```
[η ⊕ ε ⊕ σ]^{behav_buy^3}_{b2b_0003}
```
= Нейробиология × Экономика × Социология  
→ поведенческая экономика B2B-покупки (подтверждено в 3 областях)

## Расшифровка символа (алгоритм ВСЕГДА)

При любом обращении к символу — локальная модель расшифровывает:

```
1. Базовый символ → макро-область (из таблицы 32 корней)
2. Надстрочный → специализация
3. Степень → уровень подтверждённости (чем выше — тем надёжнее)
4. Подстрочный → конкретная среда/инструмент
5. Номер → конкретная ячейка в кластере
6. Проверить is_legacy → если true, предложить актуальный superseded_by
7. Проверить вектор кластера → какие решения лежат рядом
8. Сравнить last_updated → самое новое = приоритет
9. Сравнить R_f → самое используемое = проверено
10. Выдать: лучшее по (similarity × R_f × актуальность) под стек пользователя
```
