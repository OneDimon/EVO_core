# EVO-core

**Когнитивный слой между ИИ-флагманами и средами исполнения.**  
Хранит верифицированные инженерные решения как аксиомы. Выдаёт их точечно по смыслу запроса. Исключает галлюцинации и деградацию внимания при длинных проектах.

> **Правило работы с репозиторием:**  
> При входе в любую сессию — сначала этот файл. Найди свой блок, открой его карту, проверь статус сшивки со смежными блоками. Работай только внутри своего блока. Апдейти статус сшивки в обоих файлах сразу после проверки.

---

## Как читать этот файл

Каждый блок ниже — это автономная единица работы. У каждого есть:
- одна строка: что делает
- ссылка на детальную карту блока
- ссылки на смежные блоки с которыми граничит

Статусы сшивки обновляются в реальном времени в файле каждого блока.

## Порядок реализации

### Фаза 0 — Минимальное рабочее ядро
Цель: проверить что базовая идея работает. Только: запрос → поиск → декомпрессия → флагман.

```
БЛОК 02  →  БЛОК 03  →  БЛОК 01 (минимум)
```

1. [БЛОК 02](BLOCK_02_language_library.md) — PostgreSQL + pgvector + SCL-схема в коде + bootstrap 20–30 символов вручную
2. [БЛОК 03](BLOCK_03_shard_storage.md) — zstd-компрессор/декомпрессор + первые ячейки на шарде
3. [БЛОК 01](BLOCK_01_core_engine.md) — минимальный HTTP-эндпоинт: принять запрос → найти символ → декомпрессия → вернуть картридж

**Критерий выхода из Фазы 0:** 10 реальных задач прогнаны вручную, картридж выдаётся корректно.

### Фаза 1 — Полный конвейер
4. [БЛОК 06](BLOCK_06_ymm_verifier.md) — YMS-MMM верификатор
5. [БЛОК 01](BLOCK_01_core_engine.md) — апдейт: интервью флагмана + сборщик картриджа + архиватор (Тип А/Б)

### Фаза 2 — Транспорт + Иммунная система
6. [БЛОК 04](BLOCK_04_cli_layer.md) — CLI-скелетонизатор
7. [БЛОК 05](BLOCK_05_mcp_server.md) — MCP-сервер
8. [БЛОК 07](BLOCK_07_immune_system.md) — n8n + Gemini + Ollama fallback
